Why our DSL + coroutines improves standard PedroPathing

Declarative routes Paths read as movement intent instead of manual Pedro point setup and object wiring.
Coroutine cancellation `followAndIntake` can stop the active path immediately when sensors report that collection is complete.
Parallel mechanisms Drive, intake, shooter calculations, and sorter timing run together without blocking driver response.
Cleaner autonomous Autos become linear sequences like move, wait, collect, and shoot instead of large state machines.
Defensive hold `holdSuspend` uses Pedro vectors to resist pushes while scoring tasks continue in the background.
Fast tuning Callback timing, path velocity, and cycle count can be adjusted quickly between matches.

Drive

Four mecanum motors use GoBilda Pinpoint localization and PedroPathing follower control in both Autonomous and TeleOp.

The driver can switch between field-centric movement and robot-centric movement depending on match pressure.

Sorter

The v4 sorter uses side color sensors for per-slot validation and the Axon servo load wire for automatic jam detection.

Before shooting, software computes every valid slot position and chooses the closest path to minimize rotation time.

shooter render

Shooter

The v4 shooter computes trajectory from current flywheel velocity, then dynamically adjusts the hood angle instead of waiting for full speed recovery.

Camera and odometry both estimate turret angle and range, so the robot can recover alignment after contact.
